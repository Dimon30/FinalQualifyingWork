## Проблема 1 — pytest: опция `--fast` не распознавалась

**Симптом**
```
ValueError: no option named '--fast'
ERROR at setup of test_path_following[spiral_r3]
```
Все 6 тестов падали с ошибкой при запуске `pytest code/tests/`.

**Причина**  
Хук `pytest_addoption` был определён внутри тест-модуля `tests/test_curves.py`.
pytest регистрирует этот хук **только из `conftest.py`**; в тест-файле он молча игнорируется,
и опция `--fast` не попадает в пространство имён конфигурации.

**Решение (применено)**  
Перенести `pytest_addoption` из `test_curves.py` в `code/tests/conftest.py`:
```python
# code/tests/conftest.py
def pytest_addoption(parser):
    parser.addoption("--fast", action="store_true", default=False,
                     help="Ускоренный прогон: T × 0.25")
```
Удалить дублирующий блок из `test_curves.py`.

## Проблема 2 — кодировка: русский текст в консоли Windows отображался кракозябрами

**Симптом**
```
  ���������� ��������� (t = 30.0 �):
    e1           = +27.4790 �
```

**Причина**  
На Windows Python по умолчанию пишет в `stdout` через системную кодировку (cp866 или cp1251),
тогда как строки в исходнике UTF-8. Вызов `print()` с кириллицей давал мусор.

**Решение (применено)**  
В `SimResult.print_summary()` добавлено принудительное переключение `stdout` на UTF-8:
```python
# code/drone_sim/simulation/path_sim.py  →  print_summary()
import sys
try:
    if hasattr(sys.stdout, "reconfigure"):
        sys.stdout.reconfigure(encoding="utf-8")
except Exception:
    pass
```

**Альтернатива** (переменная окружения, не требует правки кода):
```bash
set PYTHONIOENCODING=utf-8
python code/scenarios/run_test.py
```

## Проблема 3 — эллиптическая спираль: дрон улетал в бесконечность

**Симптом**
```
zeta_final = -1902.8188
e2         = -1061.2651 м
||v||      = 196.4803 м/с
```

**Кривая**  
`p(s) = [4·cos(s), 2·sin(s), 0.5·s]`

**Причина**  
Наблюдатель ближайшей точки использует дискретное обновление:
```
ζ_{n+1} = ζ_n − γ · dt · H
```
Линеаризованная карта сходимости:
```
H_{n+1} = H_n · (1 − γ · dt · ‖t‖²)
```
Для устойчивости необходимо: `0 < γ · dt · ‖t‖²_max < 2`.

Для эллиптической спирали `‖t‖² = 16·sin²(s) + 4·cos²(s) + 0.25 ∈ [4.25, 16.25]`.  
Верхняя граница устойчивости: `γ < 2 / (16.25 · 0.002) ≈ 61.5`.

Изначально был задан `gamma_nearest = 1.0` (по умолчанию), затем ошибочно выставлен `120`.  
При `γ = 120`: фактор = `120 · 0.002 · 16.25 = 3.9 > 2` → наблюдатель расходится.

**Решение (применено)**  
Установить `gamma_nearest = 5.0`:
```python
cfg = SimConfig(
    ...
    gamma_nearest=5.0,   # < 2 / (||t||^2_max * dt) = 61.5
)
```
Фактор устойчивости при γ=5: `5 · 0.002 · 16.25 = 0.16` — хорошо в стороне от границы.

**Общее правило выбора gamma_nearest**
```
γ  <  2 / (‖t‖²_max · dt)

Примеры:
  спираль r=3  : ‖t‖²=10,    dt=0.002  →  γ < 100,   используем γ=1
  спираль r=2  : ‖t‖²=5,     dt=0.002  →  γ < 200,   используем γ=3
  спираль r=1.5: ‖t‖²=3.25,  dt=0.002  →  γ < 308,   используем γ=20
  эллипс [4,2] : ‖t‖²≤16.25, dt=0.002  →  γ < 61.5,  используем γ=5
```
Для неравномерных кривых (эллипс, парабола) ограничение берётся по **максимуму** `‖t‖²`.

## Проблема 4 — большой радиус окружности: дрон улетал в бесконечность

**Симптом**  
При `p(s) = [130·cos(s), 130·sin(s), 0.5·s]` и `gamma_nearest=1.0`:
```
zeta_final = -131201.9530
||v||      = 1169.2364 м/с
```

**Причина (двойная)**

1. **Нарушение условия устойчивости наблюдателя.**  
   `‖t‖² = 130² + 0.25 ≈ 16900`. При γ=1: `1 · 0.002 · 16900 = 33.8 >> 2` → взрыв.

2. **Несоответствие параметрической и дуговой скорости.**  
   При `p(s) = [r·cos(s), ...]` единица параметра s соответствует `‖t‖ = r` метрам пути.
   При r=130 и V*=1.0 регулятор задаёт скорость **130 м/с** — физически невозможно.

**Решение (подтверждено тестом)**  
Использовать **дуговую параметризацию** (делить аргумент на r):
```python
# Нормированная окружность: ||t|| = 1
curve = make_curve(lambda s: np.array([130*np.cos(s/130), 130*np.sin(s/130), 0.0]))
cfg = SimConfig(Vstar=1.0, gamma_nearest=1.0, T=60, dt=0.002, kappa=200)
```

**Результаты теста** `[130cos(s/130), 130sin(s/130), 0]`, T=60 с:
```
e1 = +0.0000 м,  e2 = -0.0026 м   ← отличная сходимость
delta_phi = -0.0003 рад
||v|| = 0.9084 м/с  (V* = 1.0)
zeta_final = 60.045             ← монотонно растёт
```
Нормировка `cos(s/r)` **работает корректно** — ‖t‖=1.000 во всех точках, контроллер сходится.

**Важно**: метод `cos(s/r)` применим только к **круговым** дугам. Для эллипса или произвольной кривой
нормировка нетривиальна и требует численного перепараметрирования — см. Проблему 5.

**Общее правило параметризации**
```
Контроллер Гл. 4 требует ‖t(s)‖ = const.

Параметризация [r·cos(s), r·sin(s)]:
  r ≤ 4   →  ‖t‖ ≈ r, допустимо; gamma_nearest считать по формуле выше
  r >> 1  →  использовать [r·cos(s/r), r·sin(s/r)]  (‖t‖ = 1)

Неравномерные кривые (эллипс, парабола) → см. Проблему 5.
```

| Кривая | Параметризация | `‖t‖` | `gamma_nearest` | Результат |
|--------|---------------|--------|-----------------|-----------|
| спираль r=3 | `[3cos(s), 3sin(s), s]` | √10 ≈ 3.16 | 1.0 | ✅ сходится |
| большой круг r=130 | `[130cos(s/130), 130sin(s/130), 0]` | 1.0 | 1.0 | ✅ сходится |
| большой круг r=130 | `[130cos(s), 130sin(s), 0]` | 130 | — | ❌ взрыв |
| эллипс | `[4cos(s), 2sin(s), 0.5s]` | 2.06..4.03 | любое | ❌ взрыв (Проблема 5) |

## Проблема 5 — эллиптическая спираль: дрон всё равно улетает (нерешено)

**Симптом**  
Даже после исправления `gamma_nearest=5.0` симуляция расходится:
```
zeta_final = -9100.5548
e2         = -438.8691 м
||v||      = 313.1663 м/с
```

**Причина (фундаментальная)**  
Контроллер вычисляет длину дуги по формуле (`path_sim.py:273`):
```python
s_arc = s * tangent_norm  # s = ζ (ближайшая точка), tangent_norm = ||t(ζ)||
```

Эта формула `s_arc = ζ · ‖t(ζ)‖` **точна только для кривых с постоянной нормой касательной** (`‖t‖ = const`). Для круговой спирали `‖t‖ = √(r²+1) = const` — работает. Для эллиптической спирали `‖t(s)‖` меняется, и производная `d(s_arc)/dζ` может уйти в отрицательную область:

```
d(s_arc)/dζ = ‖t‖ + ζ · d‖t‖/dζ

Для [4cos(s), 2sin(s), 0.5s]:  d‖t‖/dζ = 6·sin(2ζ) / ‖t‖

При ζ ≈ 3π/4:  d(s_arc)/dζ = 3.20 + 2.36 · (−1.875) = −1.23 < 0
```

`s_arc` убывает при растущем ζ → регулятор получает инвертированный знак ошибки скорости → разгоняет дрон в обратном направлении → положительная обратная связь → взрыв.

**Что НЕ помогает**  
Изменение `gamma_nearest` не устраняет проблему — взрыв происходит из-за ошибки в вычислении `s_arc`, а не из-за наблюдателя ближайшей точки.

**Необходимое исправление (не реализовано)**  
Заменить `s_arc = ζ * ‖t(ζ)‖` на настоящую длину дуги через численное интегрирование:
```python
# Вариант 1: scipy (точно, но медленно при каждом шаге)
from scipy.integrate import quad
s_arc = quad(lambda tau: np.linalg.norm(curve.t(tau)), 0.0, zeta)[0]

# Вариант 2: предвычисленная таблица arc_length(ζ) + линейная интерполяция (быстро)
```

**Затронутый файл**  
`code/drone_sim/simulation/path_sim.py`, метод `PathFollowingController._lambda_tilde_1()`, строка 273.

**Ограничение**  
До исправления контроллер корректно работает **только** на кривых с `‖t(s)‖ = const` — круговые спирали, прямые. Любая неравномерно параметризованная кривая (эллипс, парабола, произвольный путь) вызывает расходимость.
