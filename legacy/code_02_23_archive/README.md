# Проект: согласованное траекторное управление квадрокоптером (глава 4)

Это **новая** реализация на Python, ориентированная на корректное воспроизведение
модели и итогового закона управления из диссертации (глава 4):

- регулируемые переменные: `s, e1, e2, δφ`
- цель: движение вдоль гладкой пространственной траектории `S` с заданной скоростью `V*`
- реализуемый регулятор по выходу: формулы (71)–(77) / (68)–(77)  
  (динамический блок + high‑gain наблюдатель производных + sat_L)

## Структура

- `src/drone/dynamics.py` — модель объекта (как в `perfect_model_disser/chapter 4/23.slx`):
  - `a = b(φ,θ,ψ)*(u1+g) - [0,0,g]`
  - `φ̈ = u2, θ̈ = u3, ψ̈ = u4`
  - `u1` и `u2` — выходы двойных интеграторов (как в диссертации)
- `src/drone/geometry.py` — траектории и преобразование (60) в (s,e1,e2)
- `src/drone/controllers.py` — регулятор главы 4 (71)–(77) + наблюдатель (73)–(77)
  - матрица `b(θ,ψ,u1,φ)` взята **в точности** из Simulink (функция `chart_103`, `flag=full`)
- `run_ch4_demo.py` — запуск симуляции и графики

## Запуск

```bash
cd drone_control_project
python -m pip install -r requirements.txt
PYTHONPATH=src python run_ch4_demo.py
```

## Примечания

1. Поиск ближайшей точки `s = Σ(P)` сделан через численный метод Ньютона (минимизация расстояния до `η(s)`).
   Для прямой и спирали это работает стабильно и даёт корректные `s,e1,e2`.
2. В коде оставлены «крючки» под улучшение матрицы `W(·)` — сейчас используется согласованная структура,
   которая совпадает с формой, применявшейся в прошлой версии кода.
