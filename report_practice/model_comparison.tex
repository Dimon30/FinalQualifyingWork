\section{Сравнительный анализ алгоритмов оптимизации $V^*$}

\subsection{Методика тестирования}

Для объективного сравнения архитектур предсказания оптимальной скорости $V^*$ разработан модуль автоматизированного бенчмарка (\texttt{code/ml/evaluation/}).
Все модели тестируются в одинаковых условиях: одни и те же начальные состояния, параметры регулятора, продолжительность симуляции и базовая скорость $V^*_{\text{base}} = 1{,}0$.

\subsubsection*{Тестовый набор кривых}

Набор из четырёх фиксированных сценариев охватывает разные геометрические режимы:

\begin{enumerate}
    \item \textbf{Спираль $r{=}3$} — круговая спираль с параметром $p(s) = (3\cos s,\;3\sin s,\;s)^\top$, $\|t\| = \sqrt{10}$; $T{=}40$ с, $dt{=}0{,}002$ с, $\kappa{=}200$. Стандартный сценарий диссертации.
    \item \textbf{Окружность $r{=}3$, $z{=}5$} — замкнутая горизонтальная кривая, $p(s) = (3\cos s,\;3\sin s,\;5)^\top$, $\|t\|{=}3$; $T{=}40$ с.
    \item \textbf{Спираль $r{=}2$} — промежуточная кривизна, $p(s) = (2\cos s,\;2\sin s,\;s)^\top$, $\|t\| = \sqrt{5}$; $T{=}40$ с.
    \item \textbf{Прямая $x{=}s,y{=}s,z{=}s$} — пространственная прямая с аналитической функцией ближайшей точки; $T{=}30$ с, $dt{=}0{,}005$ с, $\kappa{=}100$.
\end{enumerate}

Ограничение выходного значения сети сверху (\texttt{vstar\_cap}) предотвращает нестабильность при агрессивных предсказаниях: для спиралей — $V^*_{\max} = 3{,}5$, для спирали $r{=}2$ — $4{,}0$, для прямой ограничение не задаётся.
Период прогрева (5 с для спиралей, 3 с для прямой) позволяет наблюдателю сойтись прежде, чем NN начинает влиять на управление.

\subsubsection*{Оцениваемые метрики}

\begin{itemize}
    \item $e_{1,\mathrm{RMS}}$ — RMS тангенциальной ошибки слежения, м;
    \item $e_{2,\mathrm{RMS}}$ — RMS поперечной ошибки слежения, м (основная метрика безопасности);
    \item $e_{2,\max}$ — максимальная поперечная ошибка, м;
    \item $\bar{v}$ — средняя дуговая скорость за симуляцию, м/с;
    \item Ускорение~($\times$) — отношение $\bar{v}_\mathrm{NN} / \bar{v}_\mathrm{base}$;
    \item Сошлось — $e_{2,\mathrm{RMS}} < 0{,}1$ м (порог).
\end{itemize}

\subsection{Архитектуры сравниваемых моделей}

В эксперименте участвуют четыре модели, обученные на одном CSV-датасете с одинаковым набором из семи признаков (см. раздел~\ref{sec:dataset}):

\begin{itemize}
    \item \textbf{MLP} (SpeedMLP) — полносвязная сеть 7$\to$128$\to$128$\to$64$\to$1, потери MSE, без критиков.
    \item \textbf{SAC} (SpeedSAC) — стохастический актор Gaussian($\mu$, $\sigma$) + два Q-критика; потери: отрицательная логправдоподобность (NLL) + энтропийный бонус.
    \item \textbf{TD3} (SpeedTD3) — детерминированный актор + два целевых критика с Polyak-усреднением; потери: BC MSE + Q-guided актор с задержанным обновлением.
    \item \textbf{PPO} (SpeedPPO) — Gaussian-политика + оценочная голова; потери: clipped surrogate + value MSE + entropy bonus (offline BC-PPO).
\end{itemize}

Все модели обучены при $\texttt{max\_speed} = 10{,}0$ на датасете с тем же набором кривых. Инференс унифицирован через \texttt{SpeedPredictorAny} (реестр \texttt{registry.py}).

\subsubsection*{Анализ по сценариям}

\textbf{Спираль $r{=}3$.}
Все четыре модели успешно завершили симуляцию (сошлись).
MLP даёт наименьший прирост скорости~--- 1{,}34×~--- при наибольшей $e_{2,\mathrm{RMS}} = 0{,}046$ м. SAC обеспечивает ускорение 2{,}52× при $e_{2,\mathrm{RMS}} = 0{,}014$ м. TD3 и PPO демонстрируют идентичные показатели ускорения~--- 2{,}81×, $e_{2,\mathrm{RMS}} = 0{,}014$ м~--- поскольку оба предсказывают скорость вблизи ограничения \texttt{vstar\_cap}$=3{,}5$ и rate limiter сглаживает различия в динамике.

\textbf{Окружность $r{=}3$, $z{=}5$.}
Замкнутая горизонтальная кривая с нулевым подъёмом~--- наиболее <<простой>> сценарий.
Базовый регулятор достигает $e_{2,\mathrm{RMS}} < 10^{-4}$ м.
Все модели сохраняют устойчивость; ускорение составляет 1{,}34× (MLP), 2{,}54× (SAC), 2{,}81× (TD3, PPO).

\textbf{Спираль $r{=}2$.}
Ключевой разграничивающий сценарий.
MLP, SAC и PPO остаются устойчивыми ($e_{2,\mathrm{RMS}} \leq 0{,}045$ м).
\textbf{TD3 расходится}: $e_{2,\mathrm{RMS}} = 20{,}0$ м, $e_{2,\max} = 100{,}8$ м~--- модель предсказывает V*, вплотную к ограничению $4{,}0$, что при норме касательного $\|t\| = \sqrt{5} \approx 2{,}24$ создаёт дуговую скорость $\approx 8{,}9$ м/с. Это превышает диапазон устойчивости при данных коэффициентах регулятора, и наблюдатель не успевает компенсировать быстро нарастающую ошибку.
PPO при тех же ограничениях сохраняет сходимость с ускорением 3{,}15×.

\textbf{Прямая $x{=}s,y{=}s,z{=}s$.}
На прямой нет ограничения \texttt{vstar\_cap}, поэтому агрессивные предсказания проявляются полнее.
PPO достигает наибольшего ускорения~--- 3{,}38× ($\bar{v} = 3{,}38$ м/с) при $e_{2,\mathrm{RMS}} = 0{,}070$ м.
TD3 ускоряется до 2{,}87×, однако $e_{2,\mathrm{RMS}} = 0{,}287$ м превышает порог $0{,}1$ м~--- фактически прямолинейный маршрут является <<тяжёлым>> для этой модели.
SAC показывает хороший баланс: 2{,}44×, $e_{2,\mathrm{RMS}} = 0{,}035$ м.

\subsection{Выводы}

На основании проведённого бенчмарка можно сделать следующие выводы.

\begin{enumerate}
    \item \textbf{SAC является наиболее надёжной} из трёх RL-архитектур: устойчив на всех четырёх кривых, даёт стабильное ускорение 2{,}4--2{,}5×, $e_{2,\mathrm{RMS}} \leq 0{,}035$ м. Рекомендуется как основная модель для дальнейшего использования.

    \item \textbf{PPO показывает лучшее ускорение} (до 3{,}38× на прямой) при сохранении устойчивости на всех сценариях. Несмотря на низкий $R^2 \approx -0{,}98$ при обучении (модель предсказывает значения вблизи среднего~$\approx 5{,}2$), rate limiter и safety monitor эффективно сглаживают постоянный выход, превращая его в ступенчатый разгон. Потенциал PPO реализуется при условии качественного датасета с корректным oracle-горизонтом.

    \item \textbf{TD3 нестабилен на части кривых}: расходится на спирали $r{=}2$ и превышает порог $e_{2,\mathrm{RMS}} > 0{,}1$ м на прямой. Это связано с агрессивным поведением детерминированного актора при выходе за пределы обучающего распределения. Необходима либо более жёсткое ограничение \texttt{vstar\_cap}, либо расширение датасета.

    \item \textbf{MLP наиболее консервативна} (ускорение 1{,}3×), однако всегда устойчива. Является надёжным нижним базисом.

    \item \textbf{Качество датасета критично.} Все модели обучались на датасете с коротким oracle-горизонтом (отсюда $R^2 < 0{,}3$ у TD3 и отрицательный $R^2$ у PPO). При сборе датасета с \texttt{--oracle-horizon 4000} ожидается значительное улучшение точности предсказания и, как следствие, устойчивость TD3 на всех сценариях.
\end{enumerate}
